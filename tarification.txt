Partie 2 : Les Règles de Calcul (Le Moteur de Calcul)
Chaque garantie a une "méthode de calcul". Votre application doit implémenter ces méthodes.

Méthode 1 : FIXED_AMOUNT
Logique : Retourne une valeur fixe.
Garanties associées : Défense et Recours, Avance sur recours, etc.
Calcul : Prix = Tarifs_Fixes.Prime WHERE Nom_Garantie = 'XXX'
Méthode 2 : RATE_ON_SI (Taux sur Valeur Assurée)
Logique : Applique un pourcentage à la valeur vénale du véhicule.
Garanties associées : Incendie, Vol, Vol à mains armées.
Calcul : Prix = (Taux / 100) * Vehicule.valeurs.venale
Méthode 3 : RATE_ON_NEW_VALUE (Taux sur Valeur Neuve)
Logique : Applique un pourcentage à la valeur neuve du véhicule.
Garanties associées : Bris de glaces, Tierce complète plafonnée.
Calcul : Prix = (Taux / 100) * Vehicule.valeurs.neuve
Méthode 4 : MTPL_TARIFF (Grille Responsabilité Civile)
Logique : Cherche la prime dans la grille Tarif_RC.
Garanties associées : Responsabilité Civile.
Calcul (pseudo-code) :

SELECT Prime FROM Tarif_RC
WHERE
  Categorie = Vehicule.categorie_code AND
  Energie = Vehicule.energie AND
  Vehicule.puissance_fiscale BETWEEN Puissance_Min AND Puissance_Max

Méthode 5 : TCM_TCL_MATRIX (Matrice Tierce)
Logique : Cherche un taux dans la grille Tarif_TCM_TCL puis l'applique à la valeur neuve.
Garanties associées : Tierce complète, Tierce collision.
Calcul (pseudo-code) :

// Étape 1: Trouver le taux
taux = SELECT Taux FROM Tarif_TCM_TCL
WHERE
  Categorie = Vehicule.categorie_code AND
  Type_Garantie = 'Tierce Complete' AND // ou 'Tierce Collision'
  Vehicule.valeurs.neuve BETWEEN Valeur_Neuf_Min AND Valeur_Neuf_Max AND
  Franchise = [franchise_choisie_par_utilisateur]

// Étape 2: Calculer la prime
Prix = (taux / 100) * Vehicule.valeurs.neuve

Méthode 6 : IC_IPT_FORMULA (Formule IC/IPT)
Logique : Cherche la prime dans la grille Tarif_IC_IPT.
Garanties associées : Individuelle Conducteur, Individuelle Passagers.
Calcul (pseudo-code) :

SELECT Prime FROM Tarif_IC_IPT
WHERE
  Type = 'IC' AND // ou 'IPT'
  Formule = [formule_choisie_par_utilisateur] AND
  Nb_Places = Vehicule.nb_places

Méthode 7 : CONDITIONAL_RATE (Taux Conditionnel)
Logique : Le taux à appliquer dépend d'une condition (ici, la valeur assurée).
Garanties associées : Vol, Vol à mains armées.
Calcul (pseudo-code) :

SI Vehicule.valeurs.venale <= 25000000 ALORS
  taux = 1.1 // pour Vol
SINON
  taux = 2.1 // pour Vol

Prix = (taux / 100) * Vehicule.valeurs.venale

Partie 3 : Le Processus de Calcul (L'Algorithme)
Voici le flux logique que votre application doit suivre pour générer un devis.

Entrées :

Objet Vehicule avec toutes ses caractéristiques.
Type d'offre : 'PACK' ou 'TAILOR_MADE'.
Liste des garanties choisies (si TAILOR_MADE) ou le nom du pack (si PACK).
Paramètres additionnels (ex: formule IC/IPT, montant de la franchise pour Tierce).
Algorithme :

Initialiser Total_Prime = 0.
Déterminer la liste des garanties à tarifer :
Si PACK : Récupérer la liste des garanties pré-définies pour ce pack (ex: pour "Plan EVOLUTION", la liste est RC, Défense/Recours, IC/IPT, etc.).
Si TAILOR_MADE : Utiliser la liste des garanties sélectionnées par l'utilisateur.
Boucle sur chaque Garantie dans la liste :
a. Identifier la Methode_Calcul pour cette garantie (ex: pour 'Responsabilité Civile', la méthode est MTPL_TARIFF).
b. Appliquer la règle de calcul correspondante pour obtenir Prime_Garantie.
c. Gérer les cas spéciaux :
* Si la garantie est "Défense et Recours" et que l'offre est un PACK, utiliser le Pack_Prix_Reduit de la table Tarifs_Fixes.
* Si la garantie est "Assistance", vérifier que le véhicule est bien un pickup avant d'appliquer le tarif.
d. Ajouter au total : Total_Prime = Total_Prime + Prime_Garantie.
Retourner Total_Prime.
Exemple Concret d'Implémentation
Scénario : Calculer un devis Tailor Made pour :

Véhicule : Catégorie 401, Essence, 8 CV, 5 places, Valeur Neuve=15M, Valeur Assurée=12M.
Garanties choisies : RC, Défense et Recours, Vol, Tierce Complète (avec franchise 250k).
Déroulement de l'algorithme :

Total_Prime = 0.
Liste des garanties : ['Responsabilité Civile', 'Defense et Recours', 'Vol', 'Tierce Complete'].
Boucle :
Garantie 'Responsabilité Civile':
Méthode : MTPL_TARIFF.
Calcul : Recherche dans Tarif_RC pour 401/Essence/8CV -> 102 345 FCFA.
Total_Prime = 0 + 102 345.
Garantie 'Defense et Recours':
Méthode : FIXED_AMOUNT.
Calcul : Recherche dans Tarifs_Fixes -> 7 950 FCFA.
Total_Prime = 102 345 + 7 950 = 110 295.
Garantie 'Vol':
Méthode : CONDITIONAL_RATE.
Calcul : Valeur Assurée (12M) < 25M, donc taux = 1.1%. Prime = 1.1% * 12 000 000 = 132 000 FCFA.
Total_Prime = 110 295 + 132 000 = 242 295.
Garantie 'Tierce Complete':
Méthode : TCM_TCL_MATRIX.
Calcul :
Recherche du taux dans Tarif_TCM_TCL pour 401/Tierce Complete/VN=15M/Franchise=250k -> Taux = 3.52%.
Prime = 3.52% * 15 000 000 = 528 000 FCFA.
Total_Prime = 242 295 + 528 000 = 770 295.
Résultat final : Le devis s'élève à 770 295 FCFA.


=== Implémentation (Front Admin) livrée ===

- Page: /admin/tarification pour la gestion des garanties (CRUD)
  - Fichier: src/pages/admin/AdminTarificationPage.tsx
  - Persistance: localStorage via src/features/tarification/services/guaranteeService.ts
  - Types: src/features/tarification/types.ts
- Calcul rapide (front): src/features/tarification/services/pricingService.ts
  - Additionne au prix de base: montants fixes ou % (avec min/max optionnels)
- Création d’offre (Admin > Offres):
  - Le formulaire propose la sélection de garanties par catégorie
  - Affiche un « Prix estimé avec garanties » (base + impacts)
  - La liste « Couvertures » de l’offre est alimentée avec les noms des garanties sélectionnées

Remarques
- Le champ « Prix » du formulaire d’offre est traité comme prix de base.
- La logique back-end avancée décrite dans ce fichier (grilles RC, matrices TCM/TCL, etc.) pourra remplacer/compléter le pricing front.
